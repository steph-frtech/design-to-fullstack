> Status: **design doc** ŌĆö emitter adaptation begins in Phase 28.

## What This Doc Covers

How `BackendContract` ŌåÆ Hono ~4.12 TypeScript source files.

The Hono emitter (Phase 28 adaptation of `emit-hono.ts`) reads `BackendContract` rows ŌĆö not
the Control Plane directly ŌĆö and produces the `apps/api/` package of the generated monorepo.

---

## Target Stack

| Technology | Version | Notes |
|------------|---------|-------|
| Hono | `~4.12.x` | Web Standards API (`Request`/`Response`) |
| `@hono/node-server` | matching Hono | Node.js adapter ŌĆö `serve(app)` |
| Zod | `^3.x` | Input/output validation via `@hono/zod-validator` |
| Prisma | `7.x` | DB access via `@prisma/adapter-pg` |
| Better Auth | latest stable | Auth handler mounted at `/api/auth/*` |

## Generated Output Structure

```
apps/api/
  src/
    index.ts            ŌåÉ Hono app entry: import routes, mount, serve
    auth.ts             ŌåÉ Better Auth handler (from BackendContract.auth)
    routes/
      <resource>.ts     ŌåÉ one file per Resource (CRUD family)
    operations/
      <operation>.ts    ŌåÉ one file per Operation (explicit endpoint)
      index.ts          ŌåÉ barrel re-export
    policies/
      <policy>.ts       ŌåÉ one file per Policy (middleware factory)
      index.ts
    repositories/
      <entity>.ts       ŌåÉ one file per Entity (Prisma CRUD helpers)
      index.ts
    middleware/
      session.ts        ŌåÉ session extractor middleware
      require-session.ts
      require-role.ts
    assets/
      routes.ts         ŌåÉ asset upload / serve routes
    events/
      emitter.ts        ŌåÉ typed event emitter stubs
  package.json
  tsconfig.json
  prisma/
    schema.prisma       ŌåÉ from BackendContract.schemas (DB part)
```

## index.ts ŌĆö App Entry

```ts
// Generated by dtfs codegen ŌĆö do not edit manually
import { Hono } from "hono";
import { serve } from "@hono/node-server";
import { handle } from "better-auth/node";
import { auth } from "./auth";
import customersRoutes from "./routes/customers";
import invoicesRoutes from "./routes/invoices";
import { createCustomerHandler } from "./operations/createCustomer";

const app = new Hono();

// Auth ŌĆö Better Auth handler on /api/auth/*
app.on(["GET", "POST"], "/api/auth/**", (c) => handle(auth)(c.req.raw));

// Resources
app.route("/api/customers", customersRoutes);
app.route("/api/invoices", invoicesRoutes);

// Operations
app.post("/api/operations/createCustomer", createCustomerHandler);

// Health
app.get("/health", (c) => c.json({ ok: true }));

serve({ fetch: app.fetch, port: 4000 });
export type AppType = typeof app;
```

The `AppType` export enables end-to-end type safety when the frontend SDK imports it via `hono/client`.

## routes/<resource>.ts ŌĆö CRUD Family

Generated from `BackendContract.routes` entries with `kind: "resource"`.

```ts
// Generated ŌĆö routes/customers.ts
import { Hono } from "hono";
import { zValidator } from "@hono/zod-validator";
import { requireSession } from "../middleware/require-session";
import { CustomersRepository } from "../repositories/customers";
import { CustomerInputSchema } from "../../packages/shared/src/schemas/customer";

const customers = new Hono();

// GET /api/customers
customers.get("/", requireSession, async (c) => {
  const items = await CustomersRepository.list();
  return c.json(items);
});

// GET /api/customers/:id
customers.get("/:id", requireSession, async (c) => {
  const item = await CustomersRepository.findById(c.req.param("id"));
  if (!item) return c.json({ error: "NOT_FOUND" }, 404);
  return c.json(item);
});

// POST /api/customers
customers.post("/", requireSession, zValidator("json", CustomerInputSchema), async (c) => {
  const data = c.req.valid("json");
  const item = await CustomersRepository.create(data);
  return c.json(item, 201);
});

// PATCH /api/customers/:id
customers.patch("/:id", requireSession, zValidator("json", CustomerInputSchema.partial()), async (c) => {
  const data = c.req.valid("json");
  const item = await CustomersRepository.update(c.req.param("id"), data);
  if (!item) return c.json({ error: "NOT_FOUND" }, 404);
  return c.json(item);
});

// DELETE /api/customers/:id
customers.delete("/:id", requireSession, async (c) => {
  await CustomersRepository.delete(c.req.param("id"));
  return c.json({ ok: true });
});

export default customers;
```

Routes are included only if the Resource's `exposedOps` array contains the corresponding operation.

## repositories/<entity>.ts ŌĆö Prisma CRUD Helpers

Generated from `BackendContract.schemas` entries with `kind: "entity"`.

```ts
// Generated ŌĆö repositories/customers.ts
import { prisma } from "../db";
import type { Customer } from "../../packages/shared/src";

export const CustomersRepository = {
  list: () => prisma.customer.findMany(),
  findById: (id: string) => prisma.customer.findUnique({ where: { id } }),
  create: (data: Omit<Customer, "id" | "createdAt" | "updatedAt">) =>
    prisma.customer.create({ data }),
  update: (id: string, data: Partial<Customer>) =>
    prisma.customer.update({ where: { id }, data }),
  delete: (id: string) => prisma.customer.delete({ where: { id } }),
} satisfies Record<string, (...args: unknown[]) => unknown>;
```

## RouteDescriptor ŌåÆ Handler Mapping

| RouteDescriptor.kind | Generated artifact | Pattern |
|---------------------|-------------------|---------|
| `resource` (list) | `routes/<r>.ts` GET `/` | findMany |
| `resource` (getById) | `routes/<r>.ts` GET `/:id` | findUnique |
| `resource` (create) | `routes/<r>.ts` POST `/` | create |
| `resource` (update) | `routes/<r>.ts` PATCH `/:id` | update |
| `resource` (delete) | `routes/<r>.ts` DELETE `/:id` | delete |
| `operation` | `operations/<name>.ts` POST `/api/operations/<name>` | stub handler |
| `auth` | `auth.ts` mounted at `/api/auth/**` | Better Auth handler |
| `asset` | `assets/routes.ts` | upload + serve |

## Error Handling Pattern

All routes use a consistent error response shape from `SharedContract.errors`:

```ts
// 404
return c.json({ error: "NOT_FOUND", id: c.req.param("id") }, 404);
// 403
return c.json({ error: "FORBIDDEN" }, 403);
// 422 (Zod validation ŌĆö handled by @hono/zod-validator automatically)
// 500
return c.json({ error: "SERVER_ERROR" }, 500);
```

## Generation Order (within BackendContract)

1. `prisma/schema.prisma` ŌĆö DB schema (from `BackendContract.schemas`, DB part)
2. `src/auth.ts` ŌĆö Better Auth config (from `BackendContract.auth`)
3. `src/middleware/` ŌĆö session + role middlewares
4. `src/policies/` ŌĆö policy middleware factories
5. `src/repositories/` ŌĆö Prisma CRUD helpers
6. `src/routes/` ŌĆö Resource CRUD families
7. `src/operations/` ŌĆö Operation explicit handlers
8. `src/assets/routes.ts` ŌĆö Asset routes
9. `src/events/emitter.ts` ŌĆö Event emitter stubs
10. `src/index.ts` ŌĆö App assembly + `AppType` export
