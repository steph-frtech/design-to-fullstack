## What This Doc Covers

How `BackendContract.auth` â†’ Better Auth configuration + Hono integration.

Better Auth is framework-agnostic: it exposes a single `auth.handler` that takes a standard
`Request` and returns a `Response`. The Hono adapter mounts it on `GET|POST /api/auth/**`.

---

## Target Stack

| Technology | Version | Notes |
|------------|---------|-------|
| Better Auth | latest stable | Framework-agnostic; no Hono-specific package needed |
| Hono | `~4.12.x` | Mounts Better Auth via `app.on(["GET","POST"], "/api/auth/**", ...)` |
| Prisma | `7.x` | Better Auth uses `@prisma/adapter-pg` via the shared Prisma client |
| PostgreSQL | 14+ | Better Auth tables: `user`, `session`, `account`, `verification` |

## AuthMethod â†’ BackendContract.auth

When the Control Plane contains an `AuthMethod` with `kind = SESSION` (email/password),
`compileBackendContract` writes this fragment into `BackendContract.auth`:

```jsonc
{
  "provider": "better-auth",
  "basePath": "/api/auth",
  "emailPassword": true,
  "sessionDuration": 604800,
  "tables": ["user", "session", "account", "verification"],
  "middleware": "requireSession",
  "apiKey": false,
  "bearer": false
}
```

If `AuthMethod.kind = API_KEY` is also present:

```jsonc
{
  "apiKey": true,
  "bearer": true
}
```

## Generated File: `apps/api/src/auth.ts`

```ts
// Generated by dtfs codegen â€” do not edit manually
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import { prisma } from "./db";

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "postgresql",
  }),
  emailAndPassword: {
    enabled: true,
  },
  session: {
    expiresIn: 604800, // 7 days
    updateAge: 86400,  // refresh if < 1 day remaining
  },
  basePath: "/api/auth",
});
```

If `apiKey: true`:

```ts
import { apiKey } from "better-auth/plugins";

export const auth = betterAuth({
  // ...
  plugins: [apiKey()],
});
```

## Hono Mounting (in `apps/api/src/index.ts`)

```ts
import { handle } from "better-auth/node";
import { auth } from "./auth";

// Mount Better Auth on /api/auth/**
app.on(["GET", "POST"], "/api/auth/**", (c) => handle(auth)(c.req.raw));
```

`handle` from `better-auth/node` converts a standard `Request`/`Response` pair to the
Node.js HTTP format that Hono's `@hono/node-server` expects.

## Session Middleware

The `requireSession` middleware is generated in `apps/api/src/middleware/require-session.ts`:

```ts
// Generated â€” middleware/require-session.ts
import { MiddlewareHandler } from "hono";
import { auth } from "../auth";

export const requireSession: MiddlewareHandler = async (c, next) => {
  const session = await auth.api.getSession({ headers: c.req.raw.headers });
  if (!session) {
    return c.json({ error: "UNAUTHORIZED" }, 401);
  }
  c.set("session", session.session);
  c.set("user", session.user);
  await next();
};
```

The `session` and `user` values are available in downstream handlers via `c.get("session")` and `c.get("user")`.

## Role Middleware

When `AppRole` rows exist, a role-check middleware is generated:

```ts
// Generated â€” middleware/require-role.ts
import { MiddlewareHandler } from "hono";
import type { AppRoleKey } from "../../../packages/shared/src";

export const requireRole = (role: AppRoleKey): MiddlewareHandler => async (c, next) => {
  const user = c.get("user");
  if (!user || user.role !== role) {
    return c.json({ error: "FORBIDDEN" }, 403);
  }
  await next();
};
```

## DB Tables

Better Auth manages its own tables. They are added to the Prisma schema as-is:

```prisma
model User {
  id            String    @id
  name          String
  email         String    @unique
  emailVerified Boolean
  image         String?
  createdAt     DateTime
  updatedAt     DateTime
  sessions      Session[]
  accounts      Account[]
  @@map("user")
}

model Session {
  id        String   @id
  userId    String
  token     String   @unique
  expiresAt DateTime
  ipAddress String?
  userAgent String?
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  createdAt DateTime
  updatedAt DateTime
  @@map("session")
}

model Account {
  id                    String    @id
  userId                String
  accountId             String
  providerId            String
  accessToken           String?
  refreshToken          String?
  accessTokenExpiresAt  DateTime?
  refreshTokenExpiresAt DateTime?
  scope                 String?
  idToken               String?
  password              String?
  user                  User      @relation(fields: [userId], references: [id], onDelete: Cascade)
  createdAt             DateTime
  updatedAt             DateTime
  @@map("account")
}

model Verification {
  id         String   @id
  identifier String
  value      String
  expiresAt  DateTime
  createdAt  DateTime
  updatedAt  DateTime
  @@map("verification")
}
```

These four models are always added by the emitter when `auth.emailPassword = true`.

## AuthMethod Kinds Supported

| AuthMethod.kind | Generated config |
|-----------------|-----------------|
| `SESSION` | `emailAndPassword: { enabled: true }` + session middleware |
| `API_KEY` | `plugins: [apiKey()]` + bearer middleware |
