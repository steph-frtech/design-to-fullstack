# BETTER_AUTH_GENERATION

L'emitter Better Auth (`emitAuthRuntime`) lit `BackendContract.auth` et génère `apps/api/src/auth.ts` — la configuration Better Auth — ainsi que les middlewares `require-session.ts` et `require-role.ts`. Better Auth est framework-agnostic : il expose un handler standard `Request/Response` monté sur `GET|POST /api/auth/**` dans Hono.

Liens : [[../04_Runtime_Contracts/BACKEND_CONTRACT]] · [[HONO_API_GENERATION]] · [[CLIENT_DATABASE]] · [[SHARED_SDK_GENERATION]]

## AuthMethod → BackendContract.auth

| `AuthMethod.kind` | Fragment généré |
|---|---|
| `SESSION` | `emailAndPassword: { enabled: true }` + middleware session |
| `API_KEY` | `plugins: [apiKey()]` + middleware bearer |

Fragment type SESSION :

```jsonc
{
  "provider": "better-auth",
  "basePath": "/api/auth",
  "emailPassword": true,
  "sessionDuration": 604800,
  "tables": ["user", "session", "account", "verification"],
  "middleware": "requireSession"
}
```

## apps/api/src/auth.ts généré

```ts
// Generated by dtfs codegen — do not edit manually
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import { prisma } from "./db";

export const auth = betterAuth({
  database: prismaAdapter(prisma, { provider: "postgresql" }),
  emailAndPassword: { enabled: true },
  session: { expiresIn: 604800, updateAge: 86400 },
  basePath: "/api/auth",
});
```

---

## Montage dans index.ts

```ts
import { handle } from "better-auth/node";
import { auth } from "./auth";

app.on(["GET", "POST"], "/api/auth/**", (c) => handle(auth)(c.req.raw));
```

`handle` de `better-auth/node` adapte Request/Response au format Node.js attendu par `@hono/node-server`.

## Tables DB générées dans schema.prisma

`emitPrismaSchema` ajoute automatiquement les 4 modèles Better Auth quand `auth.emailPassword = true` : `User`, `Session`, `Account`, `Verification` (voir [[CLIENT_DATABASE]] pour le détail).

## État actuel

L'emitter génère la **config stub** (`auth.ts`) mais ne génère pas encore le handler `/api/auth/*` ni les middlewares session comme barrières réelles (AUDIT_REPORT P1). Le middleware `require-session.ts` est présent mais son câblage dans les routes est partiel.
